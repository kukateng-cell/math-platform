# 幼兒至小學六年級（K-6）零基礎數學學習平台：全端開發與營運落地計畫書

> 本計畫書剔除了程式碼實作細節，將前端體驗與後台營運機制完美融合，具備高度的商業與管理視野，適合用於團隊內部對焦或向管理層／投資人進行專案匯報。

---

## 一、專案概述與核心願景

本專案旨在打造一個專為 K-6（幼兒園至小學六年級）設計的「零基礎」數學智慧學習生態系。為了確保平台不僅能在初期吸引學生，還能支撐未來海量題庫的擴展與長期的商業營運，本專案將建構兩大核心支柱：

1. **高沉浸度學習前端**：透過互動式教具、遊戲化敘事與智慧語音輔助，徹底降低低年級學生的認知門檻與數學恐懼。
2. **模組化教育數據中樞（CMS 後台）**：打造具備知識樹構建、動態參數出題與大數據監控能力的後台系統，讓內容編輯與營運團隊能獨立、高效地產出與管理教材。

---

## 二、系統架構與技術策略

本平台採用現代化全棧架構，兼顧開發效率、系統穩定性與未來擴充性，重點技術策略如下：

* **全端框架驅動**：採用現代化的 Web 框架處理前端互動與後台 API 邏輯，確保響應式設計（行動優先、平板優化）與極佳的頁面載入速度。
* **關聯式數據庫與高效讀寫**：採用強大的關聯式資料庫處理複雜的教育數據（如：多孩帳號關聯、無限層級的知識樹結構、高頻發的答題紀錄），確保數據的一致性與安全性。
* **雲端語音服務整合**：深度整合雲端智慧語音服務（TTS），提供具備童音模型的高品質發音，並透過雲端快取策略大幅降低長期的 API 調用成本。

---

## 三、用戶分層與權限控制矩陣

系統嚴格遵循兒童隱私保護規範（如 COPPA／GDPR 精神），落實「數據隔離」，並規劃明確的多重角色權限：

| 用戶角色 | 核心目標 | 核心系統權限與介面功能 |
| --- | --- | --- |
| **學生 (Student)** | 零基礎學習、沉浸式體驗 | 互動闖關場景、動態教具操作、語音引導、錯題本、成就徽章 |
| **家長 (Parent)** | 掌握進度、管理帳號 | 多孩帳號切換、學習時間限制設定、弱點雷達圖、學習報告可視化 |
| **內容編輯 (Editor)** | 教材上架、學科樹維護 | 知識樹視覺化編輯、參數化題庫管理（新增／修改／下架）、版本控制 |
| **多媒體管理員 (Media)** | 確保語音與視覺品質 | 審查自動生成的音檔、微調語音參數（語速／音調）、上傳真人覆蓋錄音 |
| **數據分析師 (Analyst)** | 平台教學質量監控 | 查閱高頻錯題警示看板、匯出匿名化與去識別化的學習數據報表 |
| **超級管理員 (Admin)** | 全域掌控與系統設定 | 用戶權限分配、系統底層參數設定、全域營運數據監控 |

## 四、核心營運中樞：後台管理系統（CMS）深度設計

後台系統是決定本教育平台能否永續規模化（Scalable）的真正核心。針對內容生產與數據追蹤，規劃以下四大自動化營運模組：

### 1. 知識圖譜與學科樹引擎

* **無級限樹狀結構**：後台提供視覺化的拖曳介面，建立「年級 → 學期 → 單元 → 知識點」的完整層級。
* **前置依賴關聯**：編輯可設定知識點的先後邏輯（例如：學習「二位數加法」前，需具備「進位概念」），為前端的自適應降級出題或補救教學提供科學演算法依據。

### 2. 參數化與模組化題庫系統

* **動態模板出題**：捨棄傳統的「靜態死題」，後台支援「變數設定」。編輯只需建立一套題目模板與數字區間，系統即可自動衍生出數千種數值變化，大幅減輕人工出題壓力並防止學生背答案。
* **專業排版與審核流**：支援中高年級的數學公式與富文本輸入，並具備草稿、審核中、已發布、已封存等嚴格的狀態流轉機制，避免編輯意外更動影響線上測驗。

### 3. 智慧語音自動化工作流

* **降本增效的自動管線**：編輯上架包含新詞彙或新題目的文本時，系統在背景自動呼叫雲端語音 API 生成音檔並永久儲存。前端呼叫時直接播放靜態檔案，實現零延遲且免重複計費。
* **雙軌覆蓋機制**：若遇到特定數學專有名詞的 AI 發音不夠自然，管理員可直接在後台手動上傳真人錄音檔進行精準覆蓋。

### 4. 營運級大數據與品質警示系統

* **高頻錯題雷達**：系統自動統計全平台各題目的錯誤率。當某道題目的錯誤率異常偏高時（例如超過設定閾值），後台會主動向編輯團隊發出警示，提示檢查題幹敘述是否有歧義或難度設定錯誤。
* **非同步數據匯出**：針對海量的學生答題紀錄，後台支援去識別化的數據匯出功能，方便資料科學團隊在不影響線上資料庫效能的前提下，進行深度的學習行為分析與演算法優化。

## 五、開發時程與落地藍圖（18 週計畫）

本計畫將專案推進分為六個階段，確保從基礎架構到高階應用的穩健交付：

### 階段一：地基與架構（第 1 - 2 週）

* 專案視覺規範制定、系統架構初始化。
* 資料庫實體關係與權限模型（RBAC）設計定案。

### 階段二：後台核心與內容管理（第 3 - 6 週）

* 開發與驗證多層級帳號權限系統。
* 實作知識樹編輯器與參數化題庫管理介面，確保內容團隊可提前進駐作業。

### 階段三：語音與多媒體管線（第 7 - 8 週）

* 串接雲端語音服務。
* 建立自動化音檔生成、雲端快取儲存與人工審核的完整工作流。

### 階段四：前端互動體驗實作（第 9 - 13 週）

* 開發 K-6 學生端敘事化、遊戲化的互動介面與專屬數學教具組件。
* 與後台題庫系統聯調，實作動態出題與學習歷程紀錄。

### 階段五：數據中樞與分析看板（第 14 - 15 週）

* 構建家長專屬的學習進度與弱點分析數據儀表板。
* 實作後台高頻錯題追蹤警示與營運數據匯出模組。

### 階段六：測試、優化與正式上線（第 16 - 18 週）

* 進行高併發壓力測試與多裝置（平板／手機）的跨屏互動相容性測試。
* 生產環境部署上線，並啟動線上錯誤追蹤與系統效能監控機制。

## 六、使用者介面需求細節（持續補充）

### 1. 登入介面

* **帳號輸入欄位**：提供使用者名稱（使用者帳號）的輸入欄位。
* **密碼輸入欄位**：提供使用者密碼的輸入欄位，輸入內容需以遮罩方式呈現以保護隱私。
* **人機驗證介面**：使用者輸入完成帳號與密碼後，於下一步進入專門的人機驗證（CAPTCHA）介面，確認操作者為真人，避免機器人惡意登入或暴力破解。

### 2. 主頁面（學習階段樹）

* **核心視覺主體**：頁面中央以一棵「學習階段樹」作為視覺主體，象徵學習歷程的成長。
* **學習階段對應**：樹的各個成長階段分別對應不同的學習階段，依序為：
  * 幼兒園
  * 小學一年級
  * 小學二年級
  * 小學三年級
  * 小學四年級
  * 小學五年級
  * 小學六年級
* **動態成長機制**：樹的大小與樣貌會隨著學習階段的提升而不斷長大，直觀呈現學習者的成長軌跡，具備激勵效果。
* **互動入口**：點擊這棵樹後，會進入（新增）另一個專屬頁面，展示該學習階段的所有學習內容。

### 3. 學習階段內容頁（分類切換）

進入特定學習階段後，此頁面承載該階段的所有學習功能。頁面上方設有一列分類切換按鍵，主要包含：

* **「做題」按鍵（預設分類）**：點擊進入該階段的答題練習區，進行數學題目練習。
* **「記錄」按鍵**：點擊後切換至練習記錄頁面（詳見下節）。

### 4. 記錄頁面（練習歷程與正確率分析）

* **最近練習記錄**：展示使用者最近幾次完成的練習記錄清單。
* **單次記錄資訊**：每筆記錄需包含該次關卡的成績與作答用時，方便回顧每次的表現。
* **正確率分析**：頁面需提供最近幾次練習的正確率分析，協助使用者（與家長）掌握學習狀況與進步趨勢。

---

> 以下為**家長端**介面需求。本平台區分為「學生端」與「家長端」兩種身份，登入後依身份進入各自的介面。家長端的核心定位為「學習管理者」，負責監督進度、分配任務與回饋意見，**不參與答題**。

### 5. 家長端主頁面（學習監督總覽）

* **身份區隔**：家長端為獨立的介面流程，與學生端的做題流程完全分離，家長端**不需要做題功能**。
* **查看學習記錄（核心功能）**：家長可在此查看其所綁定學生的完整學習記錄，掌握孩子的學習狀況與表現。

### 6. 每日學習任務管理

* **接取每日任務**：家長負責接取（領取）系統配發的每日學習任務。
* **派發任務給學生**：家長可將接取到的每日學習任務派發給指定的學生，由學生端執行完成。

### 7. 意見回饋機制

* **向管理員提出意見**：家長端提供意見回饋管道，讓家長能針對平台內容、題目或使用體驗，直接向管理員（後台營運團隊）提交意見與建議。

---

> 以下為**管理員端（Admin）**介面需求。管理員為平台的最高營運權限角色，負責維護題庫內容、管理用戶資料及管控帳號的開通與停用，確保平台內容品質與用戶秩序。

### 8. 題庫維護功能

* **修改題目權限**：管理員具備修改題庫題目的能力，可針對題幹敘述、答案或參數進行編輯與校正，以維持題目內容的正確性與品質。

### 9. 用戶資料查閱權限

* **查閱學生資料**：管理員可查看平台內所有學生的基本資料與學習相關資訊。
* **查閱家長資料**：管理員可查看平台內所有家長的基本資料與其綁定的學生關聯，便於帳號管理與問題排查。

### 10. 帳號管理功能

* **新增帳號**：管理員可視營運需求開通新增使用者帳號（學生或家長）。
* **減少（停用／刪除）帳號**：管理員可停用或刪除不再使用的帳號，維持用戶資料的整潔與安全。
